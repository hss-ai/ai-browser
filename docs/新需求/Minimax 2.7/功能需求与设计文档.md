# AI Browser — Minimax 2.7 功能需求与设计文档

## 一、项目概述

### 1.1 项目背景

AI Browser 是一款专为 AI 模型评估与对比设计的智能桌面浏览器。项目基于 Electron 框架构建，通过嵌入式 WebView 技术在统一界面内同时运行多个主流大模型，支持一键广播提问与实时横向对比。当前稳定版本为 1.4.1，已实现多模型支持、指纹伪装、代理配置等核心功能。

### 1.2 版本演进规划

本次版本代号「Minimax 2.7」，定位为一次中型升级迭代。核心目标是扩展 AI 模型支持范围、提升用户体验细节、增强数据管理能力，同时为未来的插件化架构奠定技术基础。

### 1.3 目标用户画像

本应用主要面向以下用户群体：AI 研究人员需要横向评测不同模型的输出质量；技术开发者对比各模型在代码生成、问题解答方面的表现；内容创作者寻求最佳创作辅助工具；企业用户评估 AI 解决方案的适用性。

## 二、现有功能分析

### 2.1 核心功能现状

当前版本已实现的核心功能包括以下几个方面。

**多模型并行展示**：支持在单一窗口内同时打开 ChatGPT、Claude、Gemini、DeepSeek、智谱清言五个主流 AI 模型，通过多栏布局实现实时对比。面板支持 2/3/4/5 栏自由切换，拖拽分隔线可调整各面板宽度比例。

**广播提问机制**：用户在下方的统一输入框输入问题，按 Enter 键或点击发送按钮后，系统自动将问题注入到所有可见的主面板 AI 中。广播模式可通过工具栏按钮开启或关闭，副本面板不参与广播。

**多账号副本系统**：每个 AI 模型可克隆多个独立副本，副本拥有独立的 session 存储和登录状态。副本通过独立的 partition 配置实现 cookies 和 localStorage 的隔离，适合多账号管理场景。

**指纹伪装保护**：集成 WebGL 指纹伪造、Canvas 指纹噪点注入、AudioContext 指纹修改、WebRTC 本机 IP 屏蔽等功能。每个面板和副本拥有独立的指纹配置，防止 AI 网站通过浏览器指纹进行风控检测。

**代理配置支持**：支持 HTTP、HTTPS、SOCKS4、SOCKS5 四种代理协议，可配置用户名密码认证。代理设置全局生效，应用于所有 WebView 实例。

**状态持久化**：自动保存面板布局、代理配置、指纹设置等信息，关闭重启后无需重新登录。

### 2.2 功能短板识别

经过功能评估，发现以下可改进之处。

模型覆盖不足：仅支持五个 AI 模型，缺少豆包、通义千问、文心一言、Kimi 等国内主流模型的集成，海外模型如 Grok、Mistral 等也未覆盖。

交互体验有限：缺乏对话历史记录和搜索功能，无法保存和回溯对比结果。缺少响应时间统计和 Token 消耗分析。

面板管理粗放：副本关闭需要逐个手动操作，没有批量管理能力。面板排序固定，无法自定义拖拽调整。

视觉呈现单调：虽然采用暗色主题设计，但整体视觉效果较为基础，缺少动画反馈和微交互。

可扩展性缺失：当前架构不支持插件扩展，无法动态添加新的 AI 模型或功能模块。

## 三、新增功能需求

### 3.1 扩展 AI 模型支持

**需求描述**：新增对更多主流 AI 模型的集成支持，丰富用户的可对比选项。

**新增模型列表**：

| 模型名称 | 官方网站 | 优先级 | 技术要点 |
|---------|---------|--------|---------|
| 通义千问（Qwen） | qwen.ai | P0 | 集成阿里云大模型，支持函数调用和插件系统 |
| 文心一言（ERNIE Bot） | yiyan.baidu.com | P0 | 集成百度文心，需要处理百度特殊登录流程 |
| Kimi（月之暗面） | kimi.moonshot.cn | P0 | 支持超长上下文，需适配独特的 UI 结构 |
| 豆包（ByteDance） | www.doubao.com | P1 | 字节跳动出品，需处理特殊的 Web 架构 |
| 讯飞星火（Spark） | xfyun.cn | P1 | 科大讯飞认知大模型，集成讯飞开放平台 |
| 腾讯混元（Hunyuan） | hunyuan.tencent.com | P1 | 腾讯大模型，集成微信生态能力 |

**功能要求**：新增模型需支持广播提问注入、多账号副本、指纹伪装等现有功能。新增模型配置面板，支持快速启用或禁用特定模型。

### 3.2 对话历史管理

**需求描述**：实现对话历史的本地存储、搜索和导出功能。

**功能详情**：

对话记录存储：每次广播提问自动创建一条历史记录，包含提问内容、时间戳、参与响应的 AI 模型列表。响应内容可选择是否保存，建议默认保存摘要而非完整内容以节省存储空间。

历史记录查看：侧边栏展示最近 50 条对话记录，支持按时间排序和关键词搜索。点击历史记录可查看详细对比结果。

导出功能：支持将对话历史导出为 Markdown、JSON、HTML 三种格式。导出时可选择包含或排除特定模型的响应内容。

数据管理：支持手动清理历史记录，可设置自动清理规则如保留最近 30 天或最近 100 条记录。

### 3.3 性能统计分析

**需求描述**：新增 AI 响应时间和 Token 消耗的统计功能。

**功能详情**：

响应时间统计：记录每个模型从收到提问到首次响应的时间，展示在面板状态区域。统计图表展示各模型的平均响应时间、中位数响应时间。

Token 消耗估算：通过响应内容长度估算 Token 消耗量，不同模型使用各自的估算规则。支持显示累计 Token 消耗。

响应质量评分：用户可对各模型的响应质量进行 1-5 星评分，评分数据用于后续的模型表现分析。

对比报告生成：支持生成指定时间范围内的模型表现对比报告，包含响应时间趋势、评分分布、Token 消耗汇总等信息。

### 3.4 预设问题集

**需求描述**：支持创建和管理预设问题集，便于批量测试和标准化评估。

**功能详情**：

问题集管理：创建、编辑、删除问题集，每个问题集包含多个预设问题。支持问题集导入导出。

问题分类：问题支持分类标签管理，如「编程」「写作」「数学」「常识」等。

批量执行：选择问题集后自动依次向所有启用的 AI 模型广播问题，可设置执行间隔。

执行进度：显示批量执行进度，支持暂停、继续、跳过当前问题。

### 3.5 快捷键系统升级

**需求描述**：升级快捷键系统，支持更丰富的自定义配置。

**功能详情**：

快捷键配置：提供快捷键配置界面，支持自定义各功能的触发快捷键。可配置快捷键包括发送广播、切换面板、切换布局、切换代理等。

快捷键冲突检测：配置时检测快捷键冲突并提示用户。

快捷键预览：悬浮显示各功能的快捷键提示。

宏命令支持：支持定义宏命令序列，如「发送问题后自动截图保存」。

### 3.6 主题系统

**需求描述**：扩展主题支持，提供更多视觉风格选择。

**功能详情**：

内置主题：除现有暗色主题外，新增「深夜蓝」「翠绿护眼」「经典亮色」「赛博朋克」四种主题。

主题预览：主题选择界面提供实时预览功能。

自定义主题：支持调整主色调、强调色、背景色等参数生成自定义主题。

## 四、UI/UX 改进设计

### 4.1 整体视觉升级

**设计理念**：在保持专业感的基础上，增加视觉层次和动态反馈，提升用户操作的愉悦感。

**改进要点**：

界面层次优化：重新梳理界面元素的信息层级，强化功能区、工具区、内容区的视觉区分。标题栏采用更紧凑的布局，为内容区预留更多空间。

动画系统建立：引入微交互动画系统，覆盖按钮点击、面板切换、状态变化等场景。动画时长控制在 150-300 毫秒，使用 ease-out 缓动曲线保证自然感。

图标系统升级：统一图标风格，采用线性图标设计，线宽 1.5px，圆角 2px。图标尺寸规范为工具栏 18px、面板头部 14px、状态指示 10px。

### 4.2 面板设计优化

**设计理念**：提升面板的信息密度和操作便捷性，增强各 AI 模型的品牌识别度。

**改进要点**：

面板头部优化：AI 模型 Logo 采用渐变色设计，提升品牌辨识度。模型名称旁边显示当前会话状态指示器，如「已登录」「未登录」「加载中」。

加载状态增强：面板首次加载时显示品牌色的渐变动画骨架屏。响应过程中显示品牌色的脉冲动画。

响应时间显示：在面板底部状态区域显示该模型的最近响应时间，以毫秒为单位，格式为「↑ 1.2s」。

操作菜单整合：面板头部的操作按钮整合为下拉菜单，包含刷新、复制链接、关闭面板、新增副本等选项。

### 4.3 工具栏重构

**设计理念**：优化工具栏布局，提升常用功能的可达性，减少操作步骤。

**改进要点**：

功能分组：将工具栏功能划分为三个区域。左侧为布局控制区，包含布局切换按钮。中间为核心操作区，包含广播开关和输入框。右侧为辅助功能区，包含代理、指纹、设置等按钮。

广播输入框升级：输入框支持多行文本输入，最大 5 行。支持 Markdown 预览模式。右侧发送按钮在输入非空时高亮。

快捷操作栏：广播输入框下方新增快捷操作栏，包含「清空」「粘贴板图片」「添加附件」等快捷按钮。

### 4.4 侧边栏设计

**需求描述**：新增侧边栏承载历史记录、问题集、设置等功能。

**功能详情**：

侧边栏布局：侧边栏宽度 280px，支持展开和收起。收起时显示为 48px 宽的图标栏。

功能切换：侧边栏顶部提供功能切换标签页，包括「历史记录」「问题集」「设置」三个主要功能区。

历史记录面板：展示对话历史列表，支持搜索和筛选。历史记录支持右键菜单操作，如删除、导出、重新执行等。

设置面板：提供应用设置、快捷键配置、主题选择、代理设置、指纹设置等子功能入口。

### 4.5 通知与反馈系统

**设计理念**：完善用户操作反馈系统，提供清晰的状态提示和错误信息。

**改进要点**：

通知分类：将通知分为成功、警告、错误、信息四类，使用不同的颜色和图标区分。成功使用绿色强调色，警告使用橙色，错误使用红色。

通知位置：通知显示在窗口底部居中位置，堆叠显示最多三条通知，超出自动隐藏旧通知。

错误处理：WebView 加载失败时显示品牌化的错误页面，提供重试按钮和网络诊断选项。

## 五、技术架构升级

### 5.1 模块化重构

**目标**：将现有代码重构为清晰的模块化架构，提升代码可维护性和可扩展性。

**重构方案**：

核心模块划分：将代码拆分为核心层、适配层、界面层三个层次。核心层包含状态管理、事件总线、数据持久化等基础设施。适配层包含各 AI 模型的注入逻辑封装。界面层负责 UI 渲染和用户交互。

模块通信机制：建立基于事件总线的模块间通信机制。模块通过发布订阅模式进行解耦通信，避免直接依赖。

AI 模型插件化：为新集成的 AI 模型采用插件化架构，每个模型独立的配置文件包含注入脚本、URL 配置、品牌配色等信息。添加新模型只需新增配置文件，无需修改核心代码。

### 5.2 数据存储升级

**目标**：升级数据存储方案，支持更丰富的数据结构和更大的数据量。

**技术方案**：

存储方案选择：继续使用 electron-store 作为轻量级配置存储，主要配置数据如设置、快捷键映射等保持现有方案。历史记录和对话数据迁移到 IndexedDB 存储，支持更大的数据量和更灵活的查询能力。

数据模型设计：设计规范化数据模型，对话记录包含唯一 ID、提问内容、时间戳、模型响应数组、用户评分等字段。响应内容使用压缩存储减少空间占用。

迁移脚本：提供从旧版本到新版本的数据迁移脚本，确保用户历史数据平滑过渡。

### 5.3 性能优化

**目标**：优化应用启动速度和运行时性能。

**优化措施**：

启动优化：采用懒加载策略，非核心模块延迟加载。WebView 实例按需创建，首次启动只加载默认布局的面板。

内存优化：实现 WebView 复用池机制，控制最大 WebView 实例数量。副本面板超过一定时间不活跃时自动休眠，释放内存占用。

渲染优化：使用 CSS 变量实现主题切换，减少样式重算。动画使用 transform 和 opacity 属性，利用 GPU 加速。

### 5.4 安全加固

**目标**：增强应用安全性，保护用户隐私。

**安全措施**：

敏感数据加密：代理密码、API 密钥等敏感配置使用 Electron 的 safeStorage API 进行加密存储。

CSP 配置：配置严格的 Content Security Policy，防止 XSS 攻击。

WebView 隔离：增强 WebView 隔离配置，限制对系统资源的访问能力。

### 6.2 分阶段实施计划

**第一阶段（MVP）：聚焦核心价值**

第一阶段目标为扩展 AI 模型支持和增加历史管理基础功能。时长预计两周。

工作内容包括：新增通义千问、Kimi、文心一言三个 AI 模型的集成。实现对话历史的基础存储和查看功能。进行基础 UI 优化和 Bug 修复。

**第二阶段（增强）：完善用户体验**

第二阶段目标为完善性能统计和界面优化。时长预计两周。

工作内容包括：实现响应时间和 Token 消耗统计功能。开发侧边栏组件承载历史记录和问题集入口。进行面板设计和工具栏重构。

**第三阶段（扩展）：提升可扩展性**

第三阶段目标为增强可扩展性和个性化能力。时长预计两周。

工作内容包括：完成快捷键自定义配置系统。开发主题系统和预设问题集功能。进行模块化重构为插件架构奠基。

### 6.3 风险评估与应对

**模型适配风险**：AI 网站可能频繁更新 DOM 结构导致注入脚本失效。应对措施是建立自动化 DOM 监测机制和快速响应流程。

**性能退化风险**：新增功能可能导致内存占用增加。应对措施是严格控制 WebView 实例数量和实施内存监控。

**兼容性风险**：跨平台打包可能遇到平台特有 Bug。应对措施是增加自动化测试覆盖和准备平台特定补丁。

## 七、验收标准

### 7.1 功能验收

所有 P0 优先级功能必须达到以下验收标准：新增的三个 AI 模型能够正常打开并完成登录流程。广播提问能够成功注入到新增模型。历史记录能够正确存储和回溯查看。

### 7.2 性能验收

启动时间不超过 3 秒（从点击图标到主界面完全加载）。在 5 栏布局下内存占用不超过 800MB。动画帧率保持在 50fps 以上。

### 7.3 兼容性验收

在 Windows 10/11、macOS 11+、Ubuntu 20.04+ 三个平台均能正常打包和运行。打包后的安装包体积增长不超过 20%。
