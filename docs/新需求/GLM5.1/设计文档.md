### 1.3 技术选型

| 层面 | 选型 | 理由 |
|------|------|------|
| 前端框架 | **保持 Vanilla JS + 模块化增强** | 避免大规模重写；Electron webview 与框架整合成本高 |
| 状态管理 | **自定义 EventBus + Store** | 轻量级，不引入额外依赖 |
| 数据持久化 | **better-sqlite3** | 同步 API、高性能、无外部服务 |
| 主题引擎 | **CSS Custom Properties 运行时切换** | 已有 `:root` 变量基础，扩展成本低 |
| 拖拽排序 | **自研 SortableList** | 依赖少，完全控制动画 |
| Markdown 渲染 | **marked + highlight.js** | 轻量、成熟、可注入 webview |
| 自动更新 | **electron-updater** | electron-builder 生态内置 |
| 图表 | **Chart.js** | 轻量级，评分统计够用 |

```sql
-- 评分维度
CREATE TABLE rating_dimensions (
  id          TEXT PRIMARY KEY,
```

```javascript
// theme-engine.js
class ThemeEngine {
  apply(themeId) {
    const theme = THEMES[themeId];
    const root = document.documentElement;
    Object.entries(theme.variables).forEach(([key, value]) => {
      root.style.setProperty(key, value);
    });
    localStorage.setItem('theme', themeId);
  }

  followSystem() {
    const mq = window.matchMedia('(prefers-color-scheme: dark)');
    this.apply(mq.matches ? 'dark-deep' : 'light-dawn');
    mq.addEventListener('change', (e) => {
      this.apply(e.matches ? 'dark-deep' : 'light-dawn');
    });
  }

  // 自定义主题
  saveCustom(themeId, overrides) {
    const base = THEMES['dark-deep'];
    THEMES[themeId] = { ...base, variables: { ...base.variables, ...overrides } };
  }
}
```

### 2.3 面板拖拽排序

#### 设计思路

在面板标题栏区域注册拖拽事件，通过 Flexbox `order` 属性控制顺序。

#### 核心逻辑

```javascript
// panel-sorter.js
class PanelSorter {
  constructor(container) {
    this.container = container;
    this.dragPanel = null;
    this.placeholder = null;
  }

  init() {
    // 为每个 panel header 绑定 dragstart / dragover / drop
    this.container.querySelectorAll('.panel').forEach(panel => {
      const header = panel.querySelector('.panel-header');
      header.setAttribute('draggable', true);

      header.addEventListener('dragstart', (e) => {
        this.dragPanel = panel;
        panel.classList.add('dragging');
        e.dataTransfer.effectAllowed = 'move';
      });

      header.addEventListener('dragover', (e) => {
        e.preventDefault();
        const target = this._getPanelAfter(e.clientX);
        if (target) {
          this.container.insertBefore(this.dragPanel, target);
        } else {
          this.container.appendChild(this.dragPanel);
        }
      });

      header.addEventListener('dragend', () => {
        panel.classList.remove('dragging');
        this._saveOrder();
      });
    });
  }

  _getPanelAfter(x) {
    const panels = [...this.container.querySelectorAll('.panel:not(.dragging)')];
    return panels.reduce((closest, child) => {
      const box = child.getBoundingClientRect();
      const offset = x - box.left - box.width / 2;
      if (offset < 0 && offset > closest.offset) {
        return { offset, element: child };
      }
      return closest;
    }, { offset: Number.NEGATIVE_INFINITY }).element;
  }

  _saveOrder() {
    const order = [...this.container.querySelectorAll('.panel')]
      .map(p => p.dataset.modelType);
    localStorage.setItem('panel-order', JSON.stringify(order));
  }
}
```

#### CSS 动效

```css
.panel.dragging {
  opacity: 0.5;
  transform: scale(0.95);
  transition: all 0.2s ease;
}

.panel:not(.dragging) {
  transition: transform 0.3s ease;
}
```

## 四、关键交互流程

### 4.1 广播 + 评分流程

```
用户输入 Prompt
    │
    ├─[选择模板?]──→ 模板面板 ──→ 填充变量 ──→ 回到输入框
    │
    ▼
点击发送 / Enter
    │
    ├─ 创建 Session 记录（SQLite）
    ├─ 开始计时
    ├─ 向所有可见面板注入 Prompt
    │
    ▼
各面板加载中...
    │
    ├─ 检测到首字出现 → 记录 latency
    ├─ 面板显示加载骨架屏
    │
    ▼
回复完成
    │
    ├─ 尝试抓取回复文本存入 DB
    ├─ 面板底部出现评分栏
    ├─ 用户评分（星级 + 维度标签）
    ├─ 可选标记「最佳回复」
    │
    ▼
评分数据写入 SQLite
    │
    └─ 更新统计面板排行
```

### 4.2 主题切换流程

```
用户点击设置 → 主题选择
    │
    ├─[预设主题]──→ 直接应用
    ├─[跟随系统]──→ 检测系统偏好 → 应用对应主题
    ├─[自定义]──→ 主题编辑器 → 调整颜色 → 实时预览 → 保存
    │
    ▼
ThemeEngine.apply(themeId)
    │
    ├─ 更新 CSS 变量（document.documentElement）
    ├─ 向所有 Webview 注入主题适配 CSS
    ├─ 保存选择到 localStorage + SQLite
    │
    └─ 动画过渡（300ms ease）
```

## 六、性能与兼容性考量

### 6.1 性能保障

| 关注点 | 方案 |
|--------|------|
| SQLite 查询性能 | 为 `sessions.created_at`、`sessions.starred` 建索引 |
| 历史列表渲染 | 虚拟滚动（超过 100 条时仅渲染可视区域） |
| 主题切换卡顿 | CSS 变量切换不触发重排；`transition` 限定 300ms |
| 多面板内存 | 复用 Electron session；定时清理不可见面板的缓存 |
| 导出大文件 | 流式写入；异步分片处理 |

### 6.2 数据迁移

- v1.x 用户首次启动 v2.0 时，自动创建 SQLite 数据库
- 现有 localStorage 配置（代理、指纹）自动迁移到 SQLite settings 表
- 迁移过程静默，不中断用户使用

### 6.3 向后兼容

- 新增模块均为增量引入，不修改现有模块核心逻辑
- 所有新 UI 元素（历史面板、评分栏等）默认隐藏/收起
- 用户可通过设置开关控制新功能启用状态

## 八、风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| AI 网站 DOM 结构变更导致注入失败 | 高 | 广播功能失效 | 注入脚本模块化、社区更新机制、降级为剪贴板复制 |
| SQLite 在 Electron 中编译问题 | 中 | 数据层不可用 | 备选方案: electron-store (JSON 文件) |
| Webview 跨域限制收紧 | 低 | 部分网站无法加载 | CSP 拦截保留；关注 Electron 版本更新日志 |
| 主题 CSS 变量覆盖不全 | 中 | 亮色主题部分元素不协调 | 逐步完善；CSS audit 工具检测遗漏 |
| 内存占用随历史数据增长 | 低 | 长期使用后性能下降 | 历史列表虚拟滚动；定期数据清理提醒 |
