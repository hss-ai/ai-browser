## 1. 文档目的

本文档对 AI Browser 下一阶段迭代的新需求进行系统化梳理，目标是把当前「多 AI 分屏工具」升级为「多模型对比与协作工作台」。本文档作为后续 Trellis outcomes 拆分、里程碑排期与验收评估的唯一来源。

## 2. 现状速览

### 2.1 已具备能力（v1.4.1）

- **分屏浏览**：Electron 28 + webview 同屏展示 ChatGPT / Claude / Gemini / DeepSeek / 智谱清言。
- **广播输入**：统一输入框将问题同步注入所有主面板并触发发送，回车即发。
- **多栏布局**：2 / 3 / 4 / 5 栏切换，拖拽分隔线自由调整相对宽度。
- **多账号副本**：同一 AI 可克隆独立 session 副本（`partition=persist:clone-*`）。
- **指纹伪装**：伪造 Navigator / Screen / Canvas / WebGL / AudioContext / WebRTC 指纹，每个面板独立。
- **代理配置**：支持 HTTP / HTTPS / SOCKS4 / SOCKS5，带用户名/密码。
- **登录持久化**：`persist:aisession` 持久化主面板登录状态。
- **暗色主题**：单一深色风格，紫色主色 `#7C6AF7`。

### 2.2 当前缺口

| 类别 | 主要缺口 |
| --- | --- |
| 视觉 | 无亮色/系统主题；缺少设计令牌；缺少骨架屏、微交互、响应式断点；可访问性（键盘/ARIA/对比度）未系统化 |
| AI 协作 | 广播后无法抓取响应、无法对比差异；无模板/角色、无串联工作流；无一键汇总；无导出 |
| 数据 | 无本地会话历史、无搜索；无加密；无导入导出；无备份恢复；无隐私细粒度清理 |
| 工程 | 无懒加载/休眠；无内存监控；崩溃无恢复；快捷键不可编辑；无 i18n；无自动更新；无插件机制；无命令面板 |

## 3. 设计目标与北极星指标

| 目标 | 指标（量化） |
| --- | --- |
| G1. 更美观 | 三主题覆盖率 100%；首屏 LCP < 1.5s；WCAG 2.1 AA 自动检测零阻断项 |
| G2. 更强大 | 广播后 80% 以上响应可被结构化抓取；一键汇总/导出可用；命令面板覆盖 ≥ 30 个操作 |
| G3. 更稳健 | 单面板崩溃不影响其他面板；空闲 10 分钟后内存回收 ≥ 30%；首次启动到可交互 ≤ 3s |

## 4. 用户与场景

### 4.1 典型用户

- **P1 AI 研究者 / 评测工程师**：需要同题跨模型对比答案质量与风格差异。
- **P2 产品经理 / 内容创作者**：希望将一个模型的大纲传递给另一个模型精修。
- **P3 学生 / 自学者**：希望沉淀提问历史、回看、分组整理。
- **P4 重度多账号用户**：跨账号、跨代理、跨模型切换，关注隐私与防风控。

### 4.2 核心场景

1. 同一问题一次发送，四个模型并排对比并生成差异报告。
2. 选定模型 A 作答后，自动把其答案塞给模型 B 做二次精修。
3. 历史会话按「工作 / 学习 / 生活」标签检索、导出为 Markdown。
4. 晚间接上系统深色主题，白天自动切换到亮色；高对比模式用于强光户外。

## 5. 功能需求（FR）

> 编号规则：`FR-<方向>-<序号>`；优先级：P0（必做）/ P1（重要）/ P2（可选）。

### 5.1 UI/UX 视觉升级（U）

#### FR-U01 设计令牌系统（P0）
- **背景**：现状颜色/间距/阴影散落在 `src/css/main.css`，难以主题化。
- **用户价值**：统一视觉语言，后续主题扩展零成本。
- **验收标准**：
  1. 全部色值、圆角、阴影、间距、动效时长迁移至 CSS 变量；
  2. 令牌定义文件可被 `design.md` 引用；
  3. 现有界面视觉回归零变化。

#### FR-U02 三主题（亮 / 暗 / 跟随系统）（P0）
- **背景**：当前只有暗色。
- **用户价值**：适配不同光线与用户偏好。
- **验收标准**：
  1. 标题栏提供主题切换入口；三档：Light / Dark / System；
  2. System 档实时响应 `prefers-color-scheme`；
  3. 切换无页面抖动；
  4. webview 内 AI 网站主题保持其站内设置，不被强制覆盖。

#### FR-U03 玻璃拟态标题栏与状态栏（P1）
- **验收**：开启时标题栏/状态栏具备 `backdrop-filter` 模糊 16px，禁用降级为实色；性能不低于 55 FPS。

#### FR-U04 加载骨架屏与过渡动效（P1）
- **验收**：
  1. Webview 加载期间呈现骨架屏而非空白或仅 logo；
  2. 布局切换/面板显隐采用 200ms `ease-out` 过渡；
  3. 用户可在设置中关闭动效（prefers-reduced-motion 自动生效）。

#### FR-U05 响应式断点（P1）
- **验收**：窗口宽度 < 960 px 时自动降为 2 栏；< 640 px 时自动降为 1 栏并横向滑动；状态栏 pill 溢出时横向滚动。

#### FR-U06 可访问性（P1）
- **验收**：
  1. 所有可点击元素 Tab 可达，焦点环清晰；
  2. 所有 icon-only 按钮补 `aria-label`；
  3. 主色与背景对比度 ≥ 4.5:1；提供「高对比」主题档；
  4. axe-core 自动检测 0 critical / 0 serious 问题。

#### FR-U07 图标与字体统一（P2）
- **验收**：全部图标使用同一套 stroke 风格（描边 1.5 px，圆角 2）；字体栈引入 `Inter` 与 `JetBrains Mono`（代码块）并打包进应用，不依赖外网。

### 5.2 AI 对比与协作能力（C）

#### FR-C01 响应抓取与并排展示（P0）
- **背景**：广播后无法机器读取各 AI 的答案。
- **验收**：
  1. 每个 `AI_TYPE` 新增 `extract()` 返回最后一条 assistant 文本；
  2. 广播完成后 30s 内收集到响应占比 ≥ 80%（以 DOM 稳定为准）；
  3. 提供「对比视图」抽屉，横向并排显示当次提问及各 AI 响应。

#### FR-C02 差异高亮与评分（P1）
- **验收**：
  1. 对比视图支持段落级 Diff 高亮（新增/删除/相同）；
  2. 用户可对每个回答 1–5 星评分，评分持久化；
  3. 可按「长度 / 响应时长 / 星级」排序。

#### FR-C03 一键汇总（P1）
- **验收**：
  1. 在对比视图中点击「汇总」，选择一个 AI 作为汇总者；
  2. 汇总者收到结构化 prompt：原问题 + 各 AI 答案 + 汇总指令；
  3. 汇总结果保存为一条独立记录。

#### FR-C04 角色/模板库（P1）
- **验收**：
  1. 内置 ≥ 15 个常用模板（翻译 / 润色 / 代码审查 / 学习教练 / 产品评审等）；
  2. 支持自定义模板（变量用 `{{var}}` 占位）；
  3. 模板可绑定到特定 AI（或「全部」）。

#### FR-C05 串联工作流（P2）
- **验收**：
  1. 工作流以 DAG 形式定义，节点 = AI × 模板，边传递变量；
  2. 支持从模板库一键生成常见两段式工作流（如 GPT 出大纲 → Claude 扩写）；
  3. 运行状态可视化，失败节点可重试。

#### FR-C06 多格式导出（P0）
- **验收**：
  1. 单次对话 / 对比结果 / 汇总可导出 Markdown / PDF / PNG；
  2. 导出文件含元信息（时间、模型、提问、代理/指纹标记）；
  3. 默认存储目录可自定义。

#### FR-C07 @mention 定向广播（P1）
- **验收**：输入 `@claude …` 或 `@all …` 指定目标；未匹配则走默认广播；命中时输入框有视觉提示。

### 5.3 会话与数据管理（D）

#### FR-D01 本地会话历史（P0）
- **验收**：
  1. 每个主面板与副本的提问/响应按时间线记录；
  2. 侧边抽屉以时间倒序展示，点击回填到输入框；
  3. 可按面板筛选。

#### FR-D02 标签、收藏与搜索（P1）
- **验收**：
  1. 会话支持多标签、收藏；
  2. 全文搜索延迟 < 200 ms（10000 条以内）；
  3. 搜索结果支持按 AI / 日期 / 标签二次过滤。

#### FR-D03 本地加密存储（P0）
- **验收**：
  1. 会话数据使用 Electron `safeStorage` 或 AES-256-GCM 加密落盘；
  2. 未解锁前无法在磁盘上明文看到任何历史；
  3. 解密仅在主进程完成，渲染进程不接触原始密钥。

#### FR-D04 导入 / 导出 JSON（P1）
- **验收**：整库或按筛选条件导出 JSON；导入支持冲突策略（跳过 / 覆盖 / 合并）。

#### FR-D05 云同步占位（WebDAV）（P2）
- **验收**：提供 WebDAV 配置项；上传为加密包；不强依赖第三方账号体系。

#### FR-D06 备份与一键恢复（P1）
- **验收**：
  1. 自动每日备份到本地 `backups/` 并保留最近 7 份；
  2. 设置中可一键恢复到任一备份；
  3. 备份含会话 + 设置（不含 webview session）。

#### FR-D07 隐私细粒度清理（P1）
- **验收**：设置中可按面板/副本清除 cookie / localStorage / 缓存 / indexedDB，互不影响；清理完成有确认提示。

### 5.4 高级工程与性能（E）

#### FR-E01 插件系统（P2）
- **验收**：
  1. 插件以 `manifest.json` + `index.js` 形式发布；
  2. 在独立上下文中运行，只能通过受限 API 与主应用通信；
  3. 内置插件市场入口占位；首发自研插件 ≥ 2 个（例：Token 计数、一键翻译）。

#### FR-E02 Webview 懒加载与休眠（P0）
- **验收**：
  1. 首次启动只加载默认布局内可见面板，其它按需加载；
  2. 隐藏超过 5 分钟的副本自动休眠（`about:blank`）并保留恢复标记；
  3. 再次可见时自动恢复上次 URL。

#### FR-E03 状态栏资源监控（P1）
- **验收**：状态栏显示主进程 + 所有 webview 的内存合计、CPU 占比；hover 展开每个面板明细；采样间隔可配置（默认 5s）。

#### FR-E04 崩溃自动恢复（P0）
- **验收**：
  1. 监听 webview `render-process-gone`、`unresponsive`；
  2. 弹 Toast 并允许一键重载；
  3. 最近一次提问缓存在主进程，重载后自动回填输入框。

#### FR-E05 快捷键编辑器（P1）
- **验收**：
  1. 所有命令支持自定义快捷键；
  2. 冲突检测并提示；
  3. 支持「恢复默认」。

#### FR-E06 国际化（中 / 英）（P1）
- **验收**：界面文案通过 `i18n(key)` 获取；切换语言无需重启；`zh-CN` 与 `en-US` 覆盖率 100%。

#### FR-E07 应用自动更新（P1）
- **验收**：基于 `electron-updater`；每次启动后台检查；更新有进度提示与手动触发入口；关闭后台检查的选项保留。

#### FR-E08 日志与诊断包导出（P2）
- **验收**：设置中一键导出日志 zip（含主进程日志、控制台错误、设置快照，脱敏 cookie）。

#### FR-E09 命令面板（Ctrl+K）（P0）
- **验收**：
  1. `Ctrl/Cmd + K` 呼出；
  2. 覆盖全部主要操作（切换布局 / 新增副本 / 切主题 / 打开设置 / 打开历史 / 发送模板等）；
  3. 模糊搜索，延迟 < 50 ms。

## 6. 非功能需求（NFR）

| 编号 | 维度 | 指标 |
| --- | --- | --- |
| NFR-01 | 性能 | 冷启动到可交互 ≤ 3 s；切换布局 ≤ 200 ms；命令面板输入延迟 ≤ 50 ms |
| NFR-02 | 内存 | 5 面板空闲 ≤ 1.2 GB；休眠后回收 ≥ 30% |
| NFR-03 | 可访问性 | 符合 WCAG 2.1 AA；axe-core critical=0 |
| NFR-04 | 安全 | 本地敏感数据加密；无明文落盘；`contextIsolation` 渐进恢复开启 |
| NFR-05 | 跨平台 | Windows / macOS / Linux 功能对等，安装包三端同构建 |
| NFR-06 | 可维护性 | 新模块以 ES module 组织；单文件 ≤ 400 行；关键模块单元测试覆盖 ≥ 60% |
| NFR-07 | 可观测性 | 崩溃/错误集中记录到本地日志；关键操作打点（脱敏） |
| NFR-08 | 兼容性 | 主要 AI 站点 DOM 变动时注入失败可降级为剪贴板提示（保留现有能力） |
| NFR-09 | 隐私 | 插件调用敏感 API 需权限声明；默认不发送遥测 |

## 7. 范围外（Out of Scope）

- 自研 LLM 推理能力（仍以 webview 代理官方网页为主）。
- 移动端（仅桌面）。
- 任何需要破解 AI 网站风控的非合规手段。
- 团队协作后端（多人实时协同属后续独立立项）。

## 8. 依赖与约束

- 仍使用 Electron 28（LTS 上限），渲染层优先保持原生 DOM；如引入框架须单独立项评估。
- 不得破坏现有 `persist:aisession` 与各副本 `persist:clone-*` 分区结构。
- 不得改变现有广播注入脚本的输出效果，除非经 FR-C01 重写并通过回归。
- 新增本地依赖需评估体积与许可证（见 `design.md`）。

## 10. 验收与回归策略

- 每个 FR 至少对应一份 `docs/outcomes/outcome-XX-*.md`（Trellis 流程）。
- 发布前必须通过以下回归：启动、5 面板同时加载、广播、副本克隆、代理切换、指纹切换、主题切换、崩溃重启、导出、搜索。
