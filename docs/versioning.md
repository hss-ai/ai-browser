# 版本管理规范

本项目使用语义化版本 (SemVer)。

## 版本格式

```
vMAJOR.MINOR.PATCH
```

- **MAJOR (主版本)**：破坏性更新，不兼容的 API 变更
- **MINOR (次版本)**：新功能，向后兼容
- **PATCH (修订版)**：问题修复，向后兼容

## 示例

- `v1.0.0` → 初始版本
- `v1.0.1` → 修复问题
- `v1.1.0` → 新增功能
- `v2.0.0` → 重大更新

## Git 标签

每次发布新版本时创建标签：

```bash
# 打标签
git tag v1.0.0
git push origin v1.0.0

# 查看标签
git tag -l
```

## 自动化

CI/CD 自动执行以下任务：

1. **PR/推送触发**：运行测试、lint
2. **标签推送触发**：打包全平台可执行文件并发布

详见 `.github/workflows/` 目录。
